    Section 1.7 Linear Transformations

    Up to this point, we have largely viewed matrix equations and matrix-vector multiplication through an algebraic and numerical lens. In this section, we shift our perspective to explore the graphical and geometric representations of these operations.
    🔗
    We will introduce the concept of a linear transformation, which allows us to think of matrices as dynamic functions that map vectors from one space to another. By building this concept from the ground up, we will connect our prior knowledge of functions to the new framework of vector spaces, ultimately giving us the tools to analyze rotations, reflections, and other geometric mappings.
    🔗

    Objectives

    After this section, students will be able to:
    🔗
    • verify whether a given mapping satisfies the properties of a linear transformation.
      🔗
      🔗
    • evaluate linear transformations using the principles of linearity and linear combinations.
      🔗
      🔗
    • construct the standard matrix for a linear transformation given its action on the standard basis vectors.
      🔗
      🔗
    • determine if a linear transformation is injective (one-to-one) or surjective (onto) by analyzing the pivot positions of its standard matrix.
      🔗
      🔗
    🔗
    Transformations are pretty much the same thing as functions you have learned in the past. Given that we are in MTH 261 linear algebra, the things we care about are vectors. Hence, a transformation in linear algebra is a function that takes vectors from one place and assign it to other vectors in another place.
    🔗
    Recall we learned that functions have three components: domain, range, and the rule. Similarly, transformations also have these three components. In addition, we will define a new concept (related to the range of a function), called the codomain.
    🔗
    Formally speaking, the codomain of a linear transformation is the set that includes all the possible values of a given transformation, which are called images. While this definition sounds a lot like the range of a function, there is a key difference: not everything in the codomain needs to be an image of the transformation, while everything in the range has to be some images from the transformation. Immediately, we can see that codomain should be a larger set that capture the range.
    🔗

    Note 1.7.3. But Richard... Why bother defining this special set of codomain?

    This is because in practice, we want a place to capture all the images but doesn’t necessarily need to know what the range is.
    🔗
    Here is a quick example: consider the function \(f(x) = x^2\text{.}\) We all know that the domain is \(\R\) and the range is \([0,\infty)\) instantly because we know that all number squared cannot be negative. Then we don’t need to define its codomain, another set to tell us where all possible outputs are going to live.
    🔗
    But what if my function is more complicated, like \(g(x) = x^4 - x^3 + 3\text{.}\)
    🔗
    Of course, the domain remains to be \(\R\) but we can’t quickly identify the range... All we knew, by just skimming through the function, is that outputs should also be in \(\R\text{.}\) Yet we can’t say that all real numbers can be an output. Hence, rather than calling \(\R\) the range (which is not true), we say that \(\R\) is the codomain.
    🔗
    It is totally possible for a function to have the codomain exactly the same as the range! This type of functions are called surjection (or onto transformation).
    🔗
    🔗
    The word "linear" means that the transformation must satisfy some special property. This special property is called the linearity, captured in the following definition.
    🔗

    Definition 1.7.4.

    If \(T: \R^n \to \R^m\) is a function with the following two properties:
    1. For any scalar \(c\) and any vector \(\v{x} \in \R^n\text{,}\)
      \begin{equation*} T(c\v{x}) = cT(\v{x}) \end{equation*}
      🔗
      🔗
    2. For any two vectors \(\v{x}_1, \v{x}_2 \in \R^n\text{,}\)
      \begin{equation*} T(\v{x}_1 + \v{x}_2) = T(\v{x}_1) + T(\v{x}_2) \end{equation*}
      🔗
      🔗
    then \(T\) is a linear transformation.
    🔗
    🔗
    The two properties in the definition are called the linearity. Of course, it makes sense to call a transformation "linear" if the linearity property is satisfied.
    🔗
    Note: Sometimes these two properties are combined to form one rule that looks like this:
    \begin{equation*} T(c_1 \v{x}_1 + c_2 \v{x}_2) = c_1 T(\v{x}_1) + c_2 T(\v{x}_2) \end{equation*}
    This basically tells us that we can figure out the output for a linear combination of vectors by taking the same linear combination of the outputs of those vectors.
    🔗

    Example 1.7.5.

    Determine if the following transformations are linear transformation.
    1. \(T: \R \to \R \) defined by \(T(x) = |x|\)
      🔗
      🔗
    2. \(T: \R^2 \to \R\) defined by \(T(x,y) = x + y\)
      🔗
      🔗
    3. \(T: \R^2 \to \R\) defined by \(T(x,y) = \sin(x + y)\)
      🔗
      🔗
    4. \(T: \R^2 \to \R\) defined by \(T(x,y) = 3x - 2y\)
      🔗
      🔗
    🔗
    Solution.
    1. No. It fails the scalar multiplication property. For instance, let \(c = -1\) and \(x = 5\text{.}\)
      \begin{equation*} T(-1 \cdot 5) = T(-5) = |-5| = 5 \end{equation*}
      However,
      \begin{equation*} -1 \cdot T(5) = -1 \cdot |5| = -5 \end{equation*}
      Since \(T(-5) \neq -1 \cdot T(5)\text{,}\) it is not linear.
      🔗
      🔗
    2. Yes. Let’s verify both properties. First, for any scalar \(c\text{:}\)
      \begin{equation*} T(cx, cy) = cx + cy = c(x + y) = cT(x,y) \end{equation*}
      Second, for vector addition:
      \begin{align*} T(x_1 + x_2, y_1 + y_2) \amp= (x_1 + x_2) + (y_1 + y_2) \\ \amp= (x_1 + y_1) + (x_2 + y_2) \\ \amp= T(x_1, y_1) + T(x_2, y_2) \end{align*}
      🔗
      🔗
    3. No. It fails vector addition. Let \(\v{v}_1 = (\pi/2, 0)\) and \(\v{v}_2 = (0, \pi/2)\text{.}\)
      \begin{equation*} T(\v{v}_1 + \v{v}_2) = T(\pi/2, \pi/2) = \sin(\pi/2 + \pi/2) = \sin(\pi) = 0 \end{equation*}
      But evaluating them separately gives:
      \begin{equation*} T(\v{v}_1) + T(\v{v}_2) = \sin(\pi/2 + 0) + \sin(0 + \pi/2) = 1 + 1 = 2 \end{equation*}
      🔗
      🔗
    4. Yes. Similar to part (b), we test the properties. For scalar multiplication:
      \begin{equation*} T(cx, cy) = 3(cx) - 2(cy) = c(3x - 2y) = cT(x,y) \end{equation*}
      For vector addition:
      \begin{align*} T(x_1 + x_2, y_1 + y_2) \amp= 3(x_1 + x_2) - 2(y_1 + y_2) \\ \amp= 3x_1 + 3x_2 - 2y_1 - 2y_2 \\ \amp= (3x_1 - 2y_1) + (3x_2 - 2y_2) \\ \amp= T(x_1, y_1) + T(x_2, y_2) \end{align*}
      🔗
      🔗
    🔗
    🔗
    🔗

    Checkpoint 1.7.6.

    What are some linear transformations you have learned in the past?
    🔗
    What are some non-linear transformations you have learned in the past?
    🔗
    Solution. Just some more examples of linear and non-linear transformation
    Linear Transformations:
    • Proportional relationships like \(y = kx\text{.}\)
      🔗
    • Taking the derivative of a function (Calculus!). The derivative of a sum is the sum of the derivatives, and constants pull out to the front.
      🔗
    • Evaluating definite integrals.
      🔗
    🔗
    Non-linear Transformations:
    • Quadratic functions like \(y = x^2\text{.}\)
      🔗
    • Exponential functions like \(y = e^x\text{.}\)
      🔗
    • Absolute value functions like \(y = |x|\text{.}\)
      🔗
    • Functions in the form \(y = mx + b\) where \(b \neq 0\text{.}\) This is a huge trap! In high school algebra, we call these "linear equations," but in linear algebra, they do NOT pass the test for linear transformations because \(T(cx) = m(cx) + b \neq c(mx+b)\text{.}\) We actually call these affine transformations.
      🔗
    🔗
    🔗
    🔗
    Recall in Section 1.4, every matrix defines a linear transformation in a way that
    \begin{equation*} T(\v{x}) = A\v{x} \end{equation*}
    In the next chapter, we will learn about matrix operations. Just a preview, by the Matrix Multiplication rules, we know that
    1. \(\displaystyle T(c\v{x}) = A(c\v{x}) = cA\v{x} = cT(\v{x})\)
      🔗
      🔗
    2. \(\displaystyle T(\v{x}_1 + \v{x}_2) = A(\v{x}_1 + \v{x}_2) = A\v{x}_1 + A\v{x}_2 = T(\v{x}_1) + T(\v{x}_2)\)
      🔗
      🔗
    Hence, we have the linearity in the definition of linear transformations.
    🔗

    Example 1.7.7. A non-example.

    The function \(T: \R^2 \to \R^3\) given by
    \begin{equation*} T\lp \begin{bmatrix} x \\ y \end{bmatrix} \rp = \begin{bmatrix} 2x \\ 3y \\ 5x + 7 \end{bmatrix} \end{equation*}
    is NOT a linear transformation.
    🔗
    Proof: Notice that
    \begin{equation*} T\lp \begin{bmatrix} 1 \\ 0 \end{bmatrix} \rp = \begin{bmatrix} 2 \\ 0 \\ 12 \end{bmatrix} \qquad \text{ and } \qquad T\lp \begin{bmatrix} 0 \\ 1 \end{bmatrix} \rp = \begin{bmatrix} 0 \\ 3 \\ 7 \end{bmatrix} \end{equation*}
    If this were a linear transformation, then property (ii) would mean that
    \begin{equation*} T\lp \begin{bmatrix} 1 \\ 1 \end{bmatrix} \rp = T\lp \begin{bmatrix} 1 \\ 0 \end{bmatrix} \rp + T\lp \begin{bmatrix} 0 \\ 1 \end{bmatrix} \rp = \begin{bmatrix} 2 \\ 0 \\ 12 \end{bmatrix} + \begin{bmatrix} 0 \\ 3 \\ 7 \end{bmatrix} = \begin{bmatrix} 2 \\ 3 \\ 19 \end{bmatrix} \end{equation*}
    But in fact,
    \begin{equation*} T\lp \begin{bmatrix} 1 \\ 1 \end{bmatrix} \rp = \begin{bmatrix} 2 \\ 3 \\ 12 \end{bmatrix} \end{equation*}
    Oopsie!
    🔗
    🔗

    Example 1.7.8. An example.

    The function \(T: \R^2 \to \R^2\) given by
    \begin{equation*} T\lp \begin{bmatrix} x \\ y \end{bmatrix} \rp = \begin{bmatrix} 2x + 3y \\ 5x + 7y \end{bmatrix} \end{equation*}
    IS indeed a linear transformation!
    🔗
    Proof: We will verify the two properties in the definition of linear transformations.
    1. Suppose \(c\) is a scalar and \(\begin{bmatrix} x \\ y \end{bmatrix}\) is a vector in \(\R^2\text{.}\) Then
      \begin{align*} T\lp c\begin{bmatrix} x \\ y \end{bmatrix} \rp = T\lp \begin{bmatrix} cx \\ cy \end{bmatrix} \rp \amp= \begin{bmatrix} 2(cx) + 3(cy) \\ 5(cx) + 7(cy) \end{bmatrix} \\ \amp= \begin{bmatrix} c(2x + 3y) \\ c(5x + 7y) \end{bmatrix} \\ \amp= c\begin{bmatrix} 2x + 3y \\ 5x + 7y \end{bmatrix} \\ \amp= cT\lp \begin{bmatrix} x \\ y \end{bmatrix} \rp \end{align*}
      🔗
      🔗
    2. Suppose \(\begin{bmatrix} x_1 \\ y_1 \end{bmatrix}\) and \(\begin{bmatrix} x_2 \\ y_2 \end{bmatrix}\) are two vectors in \(\R^2\text{.}\) Then
      \begin{align*} T\lp \begin{bmatrix} x_1 \\ y_1 \end{bmatrix} + \begin{bmatrix} x_2 \\ y_2 \end{bmatrix} \rp = T\lp \begin{bmatrix} x_1 + x_2 \\ y_1 + y_2 \end{bmatrix} \rp \amp= \begin{bmatrix} 2(x_1 + x_2) + 3(y_1 + y_2) \\ 5(x_1 + x_2) + 7(y_1 + y_2) \end{bmatrix} \\ \amp= \begin{bmatrix} (2x_1 + 3y_1) + (2x_2 + 3y_2) \\ (5x_1 + 7y_1) + (5x_2 + 7y_2) \end{bmatrix} \\ \amp= \begin{bmatrix} 2x_1 + 3y_1 \\ 5x_1 + 7y_1 \end{bmatrix} + \begin{bmatrix} 2x_2 + 3y_2 \\ 5x_2 + 7y_2 \end{bmatrix} \\ \amp= T\lp \begin{bmatrix} x_1 \\ y_1 \end{bmatrix} \rp + T\lp \begin{bmatrix} x_2 \\ y_2 \end{bmatrix} \rp \end{align*}
      🔗
      🔗
    Since both properties hold, we can conclude that \(T\) IS indeed a linear transformation!
    🔗
    Note: In this example, observe that the coefficients do not contribute to the proof at all (so the proof would work exactly the same even if we had different coefficients). Using the same proof, we can show that the function \(T: \R^2 \to \R^2\) given by
    \begin{equation*} T\lp \begin{bmatrix} x \\ y \end{bmatrix} \rp = \begin{bmatrix} a_1x + b_1y \\ a_2x + b_2y \end{bmatrix} \end{equation*}
    is a linear transformation for any choice of scalars \(a_1, b_1, a_2, b_2\text{.}\)
    🔗
    🔗
    Using linearity, we can quickly obtain the following theorem:
    🔗

    Theorem 1.7.9.

    If \(T\) is a linear transformation, then \(T\lp \v{0} \rp = \v{0}\)
    🔗
    🔗

    Very quick proof of this theorem.
    \(T\lp \v{0} \rp = T\lp 0 \v{u} \rp = 0 T\lp \v{u} \rp = \v{0}\)
    🔗
    🔗
    This is an important observation later when we investigate the idea of subspaces (spoiler: the kernel is a subspace and one of the reasons is because of this!)
    🔗
    The linearity property of linear transformations makes it possible to "plug in" a vector even if we don’t know the rule. That is, we can evaluate linear transformations using linear combinations.
    🔗

    Example 1.7.10.

    Suppose \(T: \R^3 \to \R^2\) is a linear transformation (but we don’t know the rule) and we do know that
    \begin{equation*} T \lp \begin{bmatrix} 1 \\ 2 \\ -1 \end{bmatrix} \rp = \begin{bmatrix} 2 \\ 0 \end{bmatrix} \qquad \text{ and } \qquad T \lp \begin{bmatrix} 1 \\ 2 \\ 3 \end{bmatrix} \rp = \begin{bmatrix} 1 \\ -4 \end{bmatrix} \end{equation*}
    Let’s say we want to evaluate \(T \lp \begin{bmatrix} 5 \\ 10 \\ 3 \end{bmatrix} \rp\text{.}\)
    🔗
    How do we do that?
    🔗
    Notice that
    \begin{equation*} \begin{bmatrix} 5 \\ 10 \\ 3 \end{bmatrix} = 3\begin{bmatrix} 1 \\ 2 \\ -1 \end{bmatrix} + 2\begin{bmatrix} 1 \\ 2 \\ 3 \end{bmatrix} \end{equation*}
    So property (i) and (ii) of linear transformations tell us that
    \begin{align*} T \lp \begin{bmatrix} 5 \\ 10 \\ 3 \end{bmatrix} \rp \amp= T \lp 3\begin{bmatrix} 1 \\ 2 \\ -1 \end{bmatrix} + 2\begin{bmatrix} 1 \\ 2 \\ 3 \end{bmatrix} \rp \\ \amp= 3T \lp \begin{bmatrix} 1 \\ 2 \\ -1 \end{bmatrix} \rp + 2T \lp \begin{bmatrix} 1 \\ 2 \\ 3 \end{bmatrix} \rp \\ \amp= 3\begin{bmatrix} 2 \\ 0 \end{bmatrix} + 2\begin{bmatrix} 1 \\ -4 \end{bmatrix} \\ \amp= \begin{bmatrix} 8 \\ -8 \end{bmatrix} \end{align*}
    🔗
    🔗
    This is a very important kind of problem! Not only does it show up on exams commonly, it will also be the key to finding the matrix for a linear transformation which is one of the most important things in linear algebra!
    🔗

    Checkpoint 1.7.11.

    Suppose \(T: \R^3 \to \R^2\) is a linear transformation and we know that
    \begin{equation*} T \lp \begin{bmatrix} 1 \\ 1 \\ 3 \end{bmatrix} \rp = \begin{bmatrix} 2 \\ 1 \end{bmatrix} \qquad \text{ and } \qquad T \lp \begin{bmatrix} 2 \\ 1 \\ 0 \end{bmatrix} \rp = \begin{bmatrix} -2 \\ 3 \end{bmatrix} \end{equation*}
    Find \(T \lp \begin{bmatrix} 1 \\ -1 \\ -9 \end{bmatrix} \rp\text{.}\)
    🔗
    Solution.
    First, we need to express \(\begin{bmatrix} 1 \\ -1 \\ -9 \end{bmatrix}\) as a linear combination of the two input vectors we know. We are looking for scalars \(c_1\) and \(c_2\) such that:
    \begin{equation*} c_1\begin{bmatrix} 1 \\ 1 \\ 3 \end{bmatrix} + c_2\begin{bmatrix} 2 \\ 1 \\ 0 \end{bmatrix} = \begin{bmatrix} 1 \\ -1 \\ -9 \end{bmatrix} \end{equation*}
    🔗
    Looking at the third row, we immediately see that \(3c_1 + 0 = -9\text{,}\) which means \(c_1 = -3\text{.}\)
    🔗
    Substituting \(c_1 = -3\) into the second row gives us \(-3(1) + c_2(1) = -1\text{,}\) which means \(c_2 = 2\text{.}\) Let’s quickly verify with the first row: \(-3(1) + 2(2) = -3 + 4 = 1\text{.}\) It works!
    🔗
    Now we can use the linearity properties to evaluate the transformation:
    \begin{align*} T\lp \begin{bmatrix} 1 \\ -1 \\ -9 \end{bmatrix} \rp \amp= T\lp -3\begin{bmatrix} 1 \\ 1 \\ 3 \end{bmatrix} + 2\begin{bmatrix} 2 \\ 1 \\ 0 \end{bmatrix} \rp \\ \amp= -3T\lp \begin{bmatrix} 1 \\ 1 \\ 3 \end{bmatrix} \rp + 2T\lp \begin{bmatrix} 2 \\ 1 \\ 0 \end{bmatrix} \rp \\ \amp= -3\begin{bmatrix} 2 \\ 1 \end{bmatrix} + 2\begin{bmatrix} -2 \\ 3 \end{bmatrix} \\ \amp= \begin{bmatrix} -6 \\ -3 \end{bmatrix} + \begin{bmatrix} -4 \\ 6 \end{bmatrix} = \begin{bmatrix} -10 \\ 3 \end{bmatrix} \end{align*}
    🔗
    🔗
    🔗
    What about the other way around? What if we are given the image of some linear transformation and we are trying to find the input?
    🔗
    Then instead of "evaluating", we will be "solving" the matrix equation. We are all experts in row reduction so this shouldn’t be a problem!
    🔗

    Example 1.7.12.

    Let \(A = \begin{bmatrix} 1 \amp 3 \\ 9 \amp -1 \\ -2 \amp 5 \end{bmatrix}\text{.}\) We define \(T: \R^2 \to \R^3\) by \(T(\v{x}) = A\v{x}\text{.}\) Now we want to find \(\v{x}\in \R^2\) such that \(T\lp \v{x} \rp = \begin{bmatrix} 3 \\ -29 \\ 16 \end{bmatrix}\text{.}\)
    🔗
    If we plug in everything we know into the transformation, then we will get the following matrix equation:
    \begin{equation*} \begin{bmatrix} 1 \amp 3 \\ 9 \amp -1 \\ -2 \amp 5 \end{bmatrix} \v{x} = \begin{bmatrix} 3 \\ -29 \\ 16 \end{bmatrix} \end{equation*}
    Converting this into an augmented matrix, we obtain
    \begin{equation*} \begin{bmatrix} 1 \amp 3 \amp \mid \amp 3 \\ 9 \amp -1 \amp \mid \amp -29 \\ -2 \amp 5 \amp \mid \amp 16 \end{bmatrix} \end{equation*}
    🔗
    Let’s apply our Gaussian Elimination skills to solve for \(\v{x}\text{:}\)
    \begin{align*} \begin{bmatrix} 1 \amp 3 \amp \mid \amp 3 \\ 9 \amp -1 \amp \mid \amp -29 \\ -2 \amp 5 \amp \mid \amp 16 \end{bmatrix} \amp\sim \begin{bmatrix} 1 \amp 3 \amp \mid \amp 3 \\ 0 \amp -28 \amp \mid \amp -56 \\ 0 \amp 11 \amp \mid \amp 22 \end{bmatrix} \amp\amp \begin{matrix} -9\text{Row 1} + \text{Row 2} \\ 2\text{Row 1} + \text{Row 3} \end{matrix} \\ \amp\sim \begin{bmatrix} 1 \amp 3 \amp \mid \amp 3 \\ 0 \amp 1 \amp \mid \amp 2 \\ 0 \amp 1 \amp \mid \amp 2 \end{bmatrix} \amp\amp \begin{matrix} -\frac{1}{28}\text{Row 2} \\ \frac{1}{11}\text{Row 3} \end{matrix} \\ \amp\sim \begin{bmatrix} 1 \amp 0 \amp \mid \amp -3 \\ 0 \amp 1 \amp \mid \amp 2 \\ 0 \amp 0 \amp \mid \amp 0 \end{bmatrix} \amp\amp \begin{matrix} -3\text{Row 2} + \text{Row 1} \\ -\text{Row 2} + \text{Row 3} \end{matrix} \end{align*}
    🔗
    Translating the reduced row echelon form back into an equation, we get \(x_1 = -3\) and \(x_2 = 2\text{.}\) Therefore, the input vector we were looking for is:
    \begin{equation*} \v{x} = \begin{bmatrix} -3 \\ 2 \end{bmatrix} \end{equation*}
    🔗
    🔗
    Notice that any vector in \(\R^2\) can be written as a linear combination of \(\v{e}_1 = \begin{bmatrix} 1 \\ 0 \end{bmatrix}\) and \(\v{e}_2 = \begin{bmatrix} 0 \\ 1 \end{bmatrix}\text{.}\)
    🔗

    Quick proof of this claim.
    Let \(\begin{bmatrix} x \\ y \end{bmatrix}\) be an arbitrary vector in \(\R^2\text{.}\) Then
    \begin{equation*} \begin{bmatrix} x \\ y \end{bmatrix} = x\begin{bmatrix} 1 \\ 0 \end{bmatrix} + y\begin{bmatrix} 0 \\ 1 \end{bmatrix} = x\v{e}_1 + y\v{e}_2 \end{equation*}
    Proved!
    🔗
    🔗
    \(\left\{ \v{e}_1, \v{e}_2 \right\}\) is called the standard basis of \(\R^2\text{.}\)
    🔗
    For \(\R^n\text{,}\) the standard basis consists of the vectors \(\left\{\v{e}_1, \v{e}_2, \ldots, \v{e}_n\right\}\) where the \(k\)-th entry of \(\v{e}_k\) is a \(1\) and all of the rest are zeros. Clearly these are all \(n\)-dimensional vectors. That means the \(\v{e}_1\) is actually a different vector depending on which \(\R^n\) you are working with.
    🔗
    For example, \(\v{e}_1\) in \(\R^2\) is \(\begin{bmatrix} 1 \\ 0 \end{bmatrix}\text{,}\) but \(\v{e}_1\) in \(\R^3\) is \(\begin{bmatrix} 1 \\ 0 \\ 0 \end{bmatrix}\text{.}\) They are NOT the same vectors!
    🔗
    Combining this fact with the two properties that every linear transformation satisfies, we can see that every linear combination defined on \(\R^2\) (this works for \(\R^n\) as well, but let’s start small!) is actually determined by what it does to the standard basis vectors \(\v{e}_1\) and \(\v{e}_2\text{.}\)
    🔗

    Quick proof of this claim.
    Let \(\v{x}\) be a vector in \(\R^2\text{.}\) Then there exist scalars \(a, b \in \R\) such that
    \begin{equation*} \v{x} = a\v{e}_1 + b\v{e}_2 \end{equation*}
    Then for any linear transformation \(T\) defined on \(\R^2\text{,}\) we have
    \begin{equation*} T(\v{x}) = T(a\v{e}_1 + b\v{e}_2) = aT(\v{e}_1) + bT(\v{e}_2) \end{equation*}
    So the output of \(T\) at \(\v{x}\) is completely determined by the output of \(T\) at \(\v{e}_1\) and \(\v{e}_2\text{.}\)
    🔗
    🔗
    We can use this fact to figure out how to find the matrix for a linear transformation (and in doing so, we really prove that such a matrix will always exist for any linear transformation)!
    🔗

    Example 1.7.13.

    Suppose that \(T: \R^2 \to \R^3\) is a linear transformation such that
    \begin{equation*} T\lp \v{e}_1 \rp = \begin{bmatrix} 2 \\ 0 \\ 3 \end{bmatrix} \qquad \text{ and } \qquad T\lp \v{e}_2 \rp = \begin{bmatrix} 1 \\ 2 \\ 3 \end{bmatrix} \end{equation*}
    What is the matrix \(A\) such that \(T(\v{x}) = A\v{x}\) for all \(\v{x} \in \R^2\text{?}\)
    🔗
    Consider any vector \(\v{x} = \begin{bmatrix} x \\ y \end{bmatrix}\) in \(\R^2\text{.}\) We know that
    \begin{align*} T\lp \begin{bmatrix} x \\ y \end{bmatrix} \rp \amp= T\lp x\begin{bmatrix} 1 \\ 0 \end{bmatrix} + y\begin{bmatrix} 0 \\ 1 \end{bmatrix} \rp \\ \amp= xT\lp \begin{bmatrix} 1 \\ 0 \end{bmatrix} \rp + yT\lp \begin{bmatrix} 0 \\ 1 \end{bmatrix} \rp \\ \amp= x\begin{bmatrix} 2 \\ 0 \\ 3 \end{bmatrix} + y\begin{bmatrix} 1 \\ 2 \\ 3 \end{bmatrix} \\ \amp= \begin{bmatrix} 2x + y \\ 2y \\ 3x + 3y \end{bmatrix} \end{align*}
    Notice that the output is what we would get if we multiplied the matrix \(\begin{bmatrix} 2 \amp 1 \\ 0 \amp 2 \\ 3 \amp 3 \end{bmatrix}\) by the vector \(\begin{bmatrix} x \\ y \end{bmatrix}\text{!}\) So
    \begin{equation*} T\lp \begin{bmatrix} x \\ y \end{bmatrix} \rp = \begin{bmatrix} 2 \amp 1 \\ 0 \amp 2 \\ 3 \amp 3 \end{bmatrix} \begin{bmatrix} x \\ y \end{bmatrix} \end{equation*}
    We found the matrix that goes with this linear transformation!
    🔗
    🔗
    Notice that in the above example, the columns of our matrix were the outputs of the standard basis vectors. That tells us exactly how to find a matrix for any linear transformation!
    🔗
    Suppose \(T: \R^n \to \R^m\) is a linear transformation. To find the matrix \(A\) such that \(T\) is given by the rule \(T(\v{x}) = A\v{x}\text{,}\) we...
    1. Figure out \(T\lp \v{e_1} \rp, T\lp \v{e_2} \rp, \ldots T\lp \v{e_n} \rp\)
      🔗
      🔗
    2. Write the matrix that has columns \(T\lp \v{e_1} \rp, T\lp \v{e_2} \rp, \ldots T\lp \v{e_n} \rp\text{:}\)
      \begin{equation*} A = \begin{bmatrix} T\lp \v{e_1} \rp \amp T\lp \v{e_2} \rp \amp \cdots \amp T\lp \v{e_n} \rp \end{bmatrix} \end{equation*}
      🔗
      🔗
    3. Pat yourself on the back
      🔗
      🔗
    Special terminology! This matrix \(A\) that we found is called the standard matrix for the linear transformation \(T\text{.}\)

    It turns out that rotations and reflections are also linear transformations!
    🔗
    Let’s find a matrix for a rotation \(\theta\) in \(\R^2\text{!}\) We need to figure out what such a rotation does to the vectors \(\begin{bmatrix} 1 \\ 0 \end{bmatrix}\) and \(\begin{bmatrix} 0 \\ 1 \end{bmatrix}\text{.}\)
    🔗
    Let’s suppose that the vector \(\begin{bmatrix} x \\ y \end{bmatrix}\) is the result of rotating \(\begin{bmatrix} 1 \\ 0 \end{bmatrix}\) counterclockwise by the angle \(\theta\text{.}\) This vector can be represented as an arrow starting at the origin and ending at the point \((x, y)\text{.}\) We can draw a triangle to figure out the exact coordinates of the vector \(\begin{bmatrix} x \\ y \end{bmatrix}\text{.}\)
    🔗
    The \(x\)-coordinate will be \(\cos(\theta)\) and the \(y\)-coordinate will be \(\sin(\theta)\text{.}\) So the rotation takes \(\begin{bmatrix} 1 \\ 0 \end{bmatrix}\) to \(\begin{bmatrix} \cos(\theta) \\ \sin(\theta) \end{bmatrix}\text{.}\)
    🔗
    If we also draw the vector \(\begin{bmatrix} 0 \\ 1 \end{bmatrix}\) and rotate it (counterclockwise) by an angle of \(\theta\text{,}\) we get a picture like this:
    🔗
    This time, the \(x\)-coordinate will be \(-\sin(\theta)\) and the \(y\)-coordinate will be \(\cos(\theta)\text{.}\) So the rotation takes \(\begin{bmatrix} 0 \\ 1 \end{bmatrix}\) to \(\begin{bmatrix} -\sin(\theta) \\ \cos(\theta) \end{bmatrix}\text{.}\)
    🔗
    Note: These pictures make it easy to see this is true when \(\theta\) is between \(0^\circ\) and \(90^\circ\text{,}\) To see that this is true regardless of how big \(\theta\) is, you can either look at cases in all of the four quadrants or you can use trig identities.
    🔗
    This means that you can find any rotation matrix by plugging \(\theta\) into
    \begin{equation*} R_\theta = \begin{bmatrix} \cos(\theta) \amp -\sin(\theta) \\ \sin(\theta) \amp \cos(\theta) \end{bmatrix} \end{equation*}
    🔗
    Now let’s find a matrix for the reflection across the line \(y = mx\text{!}\) We need fo figure out what happens to \(\begin{bmatrix} 1 \\ 0 \end{bmatrix}\) and \(\begin{bmatrix} 0 \\ 1 \end{bmatrix}\) when we reflect them across the line \(y = mx\text{.}\)
    🔗
    From analytic geometry, we know that the slope of any line perpendicular to \(y = mx\) will have slope \(-\frac{1}{m}\text{.}\) We want to find the line through \((1,0)\) that has slope \(-\frac{1}{m}\text{.}\) That is because we need to find the intersection of that line with \(y = mx\) to get the midpoint of the segment between \((1,0)\) and the point we get after reflecting \((1,0)\text{.}\)
    🔗
    That perpendicular line is given by \(y - 0 = -\frac{1}{m}(x - 1)\text{,}\) which simplifies to \(y = -\frac{1}{m}x + \frac{1}{m}\text{.}\)
    🔗
    Now let’s find the point of intersection! Substituting \(y = mx\) into this equation, we get
    \begin{equation*} mx = -\frac{1}{m}x + \frac{1}{m} \qquad \implies \qquad m^2x = -x + 1 \end{equation*}
    This means that \(x = \frac{1}{1 + m^2}\text{.}\) Then substituting again, we get \(y = m\frac{1}{1 + m^2}\text{.}\) So the midpoint between \((1,0)\) and the result of reflecting \((1,0)\) across the line \(y = mx\) is \(\lp \frac{1}{1 + m^2}, m\frac{1}{1 + m^2} \rp\text{.}\)
    🔗
    The midpoint formula tells us that the midpoint of a segment between points \((x_1, y_1)\) and \((x_2, y_2)\) is \(\lp \frac{x_1 + x_2}{2}, \frac{y_1 + y_2}{2} \rp\text{.}\) So
    \begin{equation*} \lp \frac{1}{1 + m^2}, m\frac{1}{1 + m^2} \rp = \lp \frac{1 + x}{2}, \frac{0 + y}{2} \rp \end{equation*}
    where \((x,y)\) is the point we are looking for. This gives us the equations
    \begin{equation*} \begin{cases} \frac{1}{1 + m^2} = \frac{1 + x}{2} \\ m\frac{1}{1 + m^2} = \frac{y}{2} \end{cases} \end{equation*}
    Solving these equations, we get
    \begin{equation*} \begin{cases} x = \frac{1 - m^2}{1 + m^2} \\ y = \frac{2m}{1 + m^2} \end{cases} \end{equation*}
    So the reflection takes \((1,0)\) to \(\lp \frac{1 - m^2}{1 + m^2}, \frac{2m}{1 + m^2} \rp\text{.}\)
    🔗
    A similar method can be used to find out that the reflection takes the point \((0,1)\) to the point \(\lp \frac{2m}{1 + m^2}, \frac{m^2 - 1}{1 + m^2} \rp\text{.}\)
    🔗
    Converting these to column vectors, the matrix for reflecting across the line \(y = mx\) is given by
    \begin{equation*} R_m = \begin{bmatrix} \frac{1 - m^2}{1 + m^2} \amp \frac{2m}{1 + m^2} \\ \frac{2m}{1 + m^2} \amp \frac{m^2 - 1}{1 + m^2} \end{bmatrix} \end{equation*}
    Pulling out the common factor, we can rewrite this as
    \begin{equation*} R_m = \frac{1}{1 + m^2} \begin{bmatrix} 1 - m^2 \amp 2m \\ 2m \amp m^2 - 1 \end{bmatrix} \end{equation*}

    There are some special linear transformations, and we will focus on surjection and injection:
    🔗

    Definition 1.7.17.

    A mapping \(T: \R^n \to \R^m\) is a surjection (or onto \(\R^m\)) if each \(\v{b}\in \R^m\) is the image of at least one \(\v{x} \in \R^n\text{.}\)
    🔗
    That is, every element in the codomain of \(T\) is an image of the mapping.
    🔗
    🔗

    Definition 1.7.18.

    A mapping \(T: \R^n \to \R^m\) is an injection (or one-to-one) if each \(\v{b}\in \R^m\) is the image of at most one \(\v{x}\in \R^n\text{.}\)
    🔗
    That is, a mapping \(T\) is injective if \(T(\v{x}_1) = T(\v{x}_2)\) implies \(\v{x}_1 = \v{x}_2\text{.}\)
    🔗
    🔗
    Notice how parallel each of these definitions is. Moreover, these definitions have everything to do with the existence and uniqueness of pre-images:
    • Existence: Does each \(\v{b}\in \R^m\) have a pre-image? If \(T\) is surjective, then Yes.
      🔗
      🔗
    • Uniqueness: Is each solution to \(T(\v{x}) = \v{b}\) unique? If \(T\) is injective, then Yes!
      🔗
      🔗
    🔗
    An immediate (and famous) result about injection is the following theorem:
    🔗

    Theorem 1.7.19.

    Let \(T: \R^n \to \R^n\) be a linear transformation. Then \(T\) is injective if and only if \(T(\v{x}) = \v{0}\) has only the trivial solution.
    🔗
    🔗

    Proof.
    This is a biconditional statement. Proving this statement comes in to parts:
    🔗
    Forward direction (\(\Rightarrow\)): Assume that \(T\) is injective. We want to show that the equation \(T(\v{x}) = \v{0}\) has only the trivial solution.
    🔗
    Because \(T\) is a linear transformation, we know that it maps the zero vector to the zero vector, meaning \(T(\v{0}) = \v{0}\text{.}\) Suppose there is some vector \(\v{x}\) in \(\R^n\) such that \(T(\v{x}) = \v{0}\text{.}\) Then we have:
    \begin{equation*} T(\v{x}) = T(\v{0}) \end{equation*}
    Since we assumed \(T\) is injective, the outputs being equal guarantees that the inputs must also be equal. Therefore, \(\v{x} = \v{0}\text{.}\) This shows that the trivial solution is the only solution.
    🔗
    Backward direction (\(\Leftarrow\)): Assume that \(T(\v{x}) = \v{0}\) has only the trivial solution. We want to show that \(T\) is injective.
    🔗
    Suppose there are vectors \(\v{u}\) and \(\v{v}\) in \(\R^n\) such that \(T(\v{u}) = T(\v{v})\text{.}\) Subtracting \(T(\v{v})\) from both sides, we have
    \begin{equation*} T(\v{u}) - T(\v{v}) = \v{0} \end{equation*}
    By the linearity properties, this implies that
    \begin{equation*} T(\v{u} - \v{v}) = \v{0} \end{equation*}
    By our initial assumption, the only vector that maps to the zero vector is the zero vector itself. This forces the input to be zero. That is,
    \begin{equation*} \v{u} - \v{v} = \v{0} \end{equation*}
    This shows that \(\v{u} = \v{v}\text{,}\) and hence \(T\) is injective.
    🔗
    What we have established that these two claims are either both true or both false (we assume one of them is true, and show the other one is also true). Another way of calling biconditional statements are equivalence statement.
    🔗
    🔗
    Last but not least, let’s make a connection between these two types of the special transformations and what we have learned in the past through the following two statements!
    🔗

    Theorem 1.7.20.

    Let \(T: \R^n \to \R^m\) be a linear transformation with standard matrix \(A\text{.}\) Then \(T\) is surjective if and only if the columns of \(A\) span the codomain, \(\R^m\text{.}\)
    🔗
    🔗

    Proof.
    Observe that the theorem is a biconditional statement. But instead of proving both directions, let’s do a slick trick!
    🔗
    Notation clarification! The symbol \(\implies\) means implication!
    🔗
    Let’s consider the forward direction. The chain of implication looks like this:
    \begin{align*} \amp T \text{ is surjective} \\ \implies \qquad \amp \forall \v{b}\in \R^m, T(\v{x}) = \v{b} \text{ has at least one solution} \\ \implies \qquad \amp \forall \v{b}\in \R^m, A\v{x} = \v{b} \text{ is consistent} \\ \implies \qquad \amp \text{The columns of } A \text{ span } \R^m \end{align*}
    Okay. And this is supposed to be impressive how?
    🔗
    Observe that the statements here are all equivalent! That is, to prove the other direction, we can just reverse all the implication arrows and the argument stays true! So the proof of this theorem is to change all the implication arrow to a biconditional arrow!
    \begin{align*} \amp T \text{ is surjective} \\ \iff \qquad \amp \forall \v{b}\in \R^m, T(\v{x}) = \v{b} \text{ has at least one solution} \\ \iff \qquad \amp \forall \v{b}\in \R^m, A\v{x} = \v{b} \text{ is consistent} \\ \iff \qquad \amp \text{The columns of } A \text{ span } \R^m \end{align*}
    🔗
    🔗

    Theorem 1.7.21.

    Let \(T: \R^n \to \R^m\) be a linear transformation with standard matrix \(A\text{.}\) Then \(T\) is injective if and only if the columns of \(A\) are linearly independent.
    🔗
    🔗

    Proof.
    We can prove this theorem using the same slick trick!
    \begin{align*} \amp T \text{ is injective} \\ \iff \qquad \amp T(\v{x}) = \v{0} \text{ has has only the trivial solution} \\ \iff \qquad \amp A\v{x} = \v{0} \text{ has only the trivial solution} \\ \iff \qquad \amp \text{The columns of } A \text{ are linearly independent } \end{align*}
    🔗
    🔗

    Example 1.7.22.

    Let \(T: \R^2 \to \R^3\) defined by
    \begin{equation*} T\lp \begin{bmatrix} x_1 \\ x_2 \end{bmatrix} \rp = \begin{bmatrix} 3x_1 + x_2 \\ 5x_1 + 7x_2 \\ x_1 + 3x_2 \end{bmatrix} \end{equation*}
    Is \(T\) a linear transformation? Is it injective? Is it surjective?
    🔗
    First, we determine if \(T\) is a linear transformation. We can "factor out" the variables to rewrite the transformation as a matrix-vector product:
    \begin{equation*} T(\v{x}) = \begin{bmatrix} 3 \amp 1 \\ 5 \amp 7 \\ 1 \amp 3 \end{bmatrix} \begin{bmatrix} x_1 \\ x_2 \end{bmatrix} \end{equation*}
    Let \(A\) be this \(3 \times 2\) matrix. Because \(T(\v{x}) = A\v{x}\text{,}\) it is a matrix transformation. Every matrix transformation is a linear transformation, so yes, \(T\) is a linear transformation.
    🔗
    Next, we check if \(T\) is injective. By our previous theorem, \(T\) is injective if and only if the homogeneous equation \(T(\v{x}) = \v{0}\) has only the trivial solution. To determine this, we find the reduced row echelon form of the standard matrix \(A\text{:}\)
    \begin{equation*} \begin{bmatrix} 3 \amp 1 \\ 5 \amp 7 \\ 1 \amp 3 \end{bmatrix} \sim \begin{bmatrix} 1 \amp 3 \\ 5 \amp 7 \\ 3 \amp 1 \end{bmatrix} \sim \begin{bmatrix} 1 \amp 3 \\ 0 \amp -8 \\ 0 \amp -8 \end{bmatrix} \sim \begin{bmatrix} 1 \amp 0 \\ 0 \amp 1 \\ 0 \amp 0 \end{bmatrix} \end{equation*}
    Because there is a pivot in every column, there are no free variables. The equation \(A\v{x} = \v{0}\) has only the trivial solution, \(\v{x} = \v{0}\text{.}\) Therefore, \(T\) is injective.
    🔗
    Finally, we determine if \(T\) is surjective. The transformation \(T\) is surjective if and only if its columns span the codomain, \(\R^3\text{.}\) This occurs if and only if there is a pivot position in every row of \(A\text{.}\) Looking at the reduced row echelon form of \(A\text{,}\) the third row does not contain a pivot. Consequently, the columns of \(A\) do not span \(\R^3\text{,}\) meaning there are vectors in the codomain that cannot be reached. Thus, \(T\) is not surjective.
