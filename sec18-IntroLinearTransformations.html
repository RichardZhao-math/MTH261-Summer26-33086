    Section 1.7 Introduction to Linear Transformations

    Up to this point, we have largely viewed matrix equations and matrix-vector multiplication through an algebraic and numerical lens. In this section, we shift our perspective to explore the graphical and geometric representations of these operations.
    🔗
    We will introduce the concept of a linear transformation, which allows us to think of matrices as dynamic functions that map vectors from one space to another.
    🔗

    Objectives

    After this section, students will be able to:
    🔗
    • verify whether a given mapping satisfies the properties of a linear transformation.
      🔗
      🔗
    • evaluate the image of a linear transformation and solve the pre-image of a linear transformation.
      🔗
      🔗
    • evaluate linear transformations using the principles of linearity and linear combinations.
      🔗
      🔗
    🔗
    Transformations are pretty much the same thing as functions you have learned in the past. Given that we are in MTH 261 linear algebra, the things we care about are vectors. Hence, a transformation in linear algebra is a function that takes vectors from one place and assign it to other vectors in another place.
    🔗
    Recall we learned that functions have three components: domain, range, and the rule. Similarly, transformations also have these three components. In addition, we will define a new concept (related to the range of a function), called the codomain.
    🔗
    Formally speaking, the codomain of a linear transformation is the set that includes all the possible values of a given transformation, which are called images. While this definition sounds a lot like the range of a function, there is a key difference: not everything in the codomain needs to be an image of the transformation, while everything in the range has to be some images from the transformation. Immediately, we can see that codomain should be a larger set that capture the range.
    🔗

    Note 1.7.3. But Richard... Why bother defining this special set of codomain?

    This is because in practice, we want a place to capture all the images but doesn’t necessarily need to know what the range is.
    🔗
    Here is a quick example: consider the function \(f(x) = x^2\text{.}\) We all know that the domain is \(\R\) and the range is \([0,\infty)\) instantly because we know that all number squared cannot be negative. Then we don’t need to define its codomain, another set to tell us where all possible outputs are going to live.
    🔗
    But what if my function is more complicated, like \(g(x) = x^4 - x^3 + 3\text{.}\)
    🔗
    Of course, the domain remains to be \(\R\) but we can’t quickly identify the range... All we knew, by just skimming through the function, is that outputs should also be in \(\R\text{.}\) Yet we can’t say that all real numbers can be an output. Hence, rather than calling \(\R\) the range (which is not true), we say that \(\R\) is the codomain.
    🔗
    It is totally possible for a function to have the codomain exactly the same as the range! This type of functions are called surjection (or onto transformation).
    🔗
    🔗
    The word "linear" means that the transformation must satisfy some special property. This special property is called the linearity, captured in the following definition.
    🔗

    Definition 1.7.4.

    If \(T: \R^n \to \R^m\) is a function with the following two properties:
    1. For any scalar \(c\) and any vector \(\v{x} \in \R^n\text{,}\)
      \begin{equation*} T(c\v{x}) = cT(\v{x}) \end{equation*}
      🔗
      🔗
    2. For any two vectors \(\v{x}_1, \v{x}_2 \in \R^n\text{,}\)
      \begin{equation*} T(\v{x}_1 + \v{x}_2) = T(\v{x}_1) + T(\v{x}_2) \end{equation*}
      🔗
      🔗
    then \(T\) is a linear transformation.
    🔗
    🔗
    The two properties in the definition are called the linearity. Of course, it makes sense to call a transformation "linear" if the linearity property is satisfied.
    🔗
    Note: Sometimes these two properties are combined to form one rule that looks like this:
    \begin{equation*} T(c_1 \v{x}_1 + c_2 \v{x}_2) = c_1 T(\v{x}_1) + c_2 T(\v{x}_2) \end{equation*}
    This basically tells us that we can figure out the output for a linear combination of vectors by taking the same linear combination of the outputs of those vectors.
    🔗

    Example 1.7.5.

    Determine if the following transformations are linear transformation.
    1. \(T: \R \to \R \) defined by \(T(x) = |x|\)
      🔗
      🔗
    2. \(T: \R^2 \to \R\) defined by \(T(x,y) = x + y\)
      🔗
      🔗
    3. \(T: \R^2 \to \R\) defined by \(T(x,y) = \sin(x + y)\)
      🔗
      🔗
    4. \(T: \R^2 \to \R\) defined by \(T(x,y) = 3x - 2y\)
      🔗
      🔗
    🔗
    Solution.
    1. No. It fails the scalar multiplication property. For instance, let \(c = -1\) and \(x = 5\text{.}\)
      \begin{equation*} T(-1 \cdot 5) = T(-5) = |-5| = 5 \end{equation*}
      However,
      \begin{equation*} -1 \cdot T(5) = -1 \cdot |5| = -5 \end{equation*}
      Since \(T(-5) \neq -1 \cdot T(5)\text{,}\) it is not linear.
      🔗
      🔗
    2. Yes. Let’s verify both properties. First, for any scalar \(c\text{:}\)
      \begin{equation*} T(cx, cy) = cx + cy = c(x + y) = cT(x,y) \end{equation*}
      Second, for vector addition:
      \begin{align*} T(x_1 + x_2, y_1 + y_2) \amp= (x_1 + x_2) + (y_1 + y_2) \\ \amp= (x_1 + y_1) + (x_2 + y_2) \\ \amp= T(x_1, y_1) + T(x_2, y_2) \end{align*}
      🔗
      🔗
    3. No. It fails vector addition. Let \(\v{v}_1 = (\pi/2, 0)\) and \(\v{v}_2 = (0, \pi/2)\text{.}\)
      \begin{equation*} T(\v{v}_1 + \v{v}_2) = T(\pi/2, \pi/2) = \sin(\pi/2 + \pi/2) = \sin(\pi) = 0 \end{equation*}
      But evaluating them separately gives:
      \begin{equation*} T(\v{v}_1) + T(\v{v}_2) = \sin(\pi/2 + 0) + \sin(0 + \pi/2) = 1 + 1 = 2 \end{equation*}
      🔗
      🔗
    4. Yes. Similar to part (b), we test the properties. For scalar multiplication:
      \begin{equation*} T(cx, cy) = 3(cx) - 2(cy) = c(3x - 2y) = cT(x,y) \end{equation*}
      For vector addition:
      \begin{align*} T(x_1 + x_2, y_1 + y_2) \amp= 3(x_1 + x_2) - 2(y_1 + y_2) \\ \amp= 3x_1 + 3x_2 - 2y_1 - 2y_2 \\ \amp= (3x_1 - 2y_1) + (3x_2 - 2y_2) \\ \amp= T(x_1, y_1) + T(x_2, y_2) \end{align*}
      🔗
      🔗
    🔗
    🔗
    🔗

    Checkpoint 1.7.6.

    What are some linear transformations you have learned in the past?
    🔗
    What are some non-linear transformations you have learned in the past?
    🔗
    Solution. Just some more examples of linear and non-linear transformation
    Linear Transformations:
    • Proportional relationships like \(y = kx\text{.}\)
      🔗
    • Taking the derivative of a function (Calculus!). The derivative of a sum is the sum of the derivatives, and constants pull out to the front.
      🔗
    • Evaluating definite integrals.
      🔗
    🔗
    Non-linear Transformations:
    • Quadratic functions like \(y = x^2\text{.}\)
      🔗
    • Exponential functions like \(y = e^x\text{.}\)
      🔗
    • Absolute value functions like \(y = |x|\text{.}\)
      🔗
    • Functions in the form \(y = mx + b\) where \(b \neq 0\text{.}\) This is a huge trap! In high school algebra, we call these "linear equations," but in linear algebra, they do NOT pass the test for linear transformations because \(T(cx) = m(cx) + b \neq c(mx+b)\text{.}\) We actually call these affine transformations.
      🔗
    🔗
    🔗
    🔗
    It turns out that every matrix defines a linear transformation in a way that
    \begin{equation*} T(\v{x}) = A\v{x} \end{equation*}
    In the next chapter, we will learn about matrix operations. Just a preview, by the Matrix Multiplication rules, we know that
    1. \(\displaystyle T(c\v{x}) = A(c\v{x}) = cA\v{x} = cT(\v{x})\)
      🔗
      🔗
    2. \(\displaystyle T(\v{x}_1 + \v{x}_2) = A(\v{x}_1 + \v{x}_2) = A\v{x}_1 + A\v{x}_2 = T(\v{x}_1) + T(\v{x}_2)\)
      🔗
      🔗
    Hence, we have the linearity in the definition of linear transformations.
    🔗

    Example 1.7.7. A non-example.

    The function \(T: \R^2 \to \R^3\) given by
    \begin{equation*} T\lp \begin{bmatrix} x \\ y \end{bmatrix} \rp = \begin{bmatrix} 2x \\ 3y \\ 5x + 7 \end{bmatrix} \end{equation*}
    is NOT a linear transformation.
    🔗
    Proof: Notice that
    \begin{equation*} T\lp \begin{bmatrix} 1 \\ 0 \end{bmatrix} \rp = \begin{bmatrix} 2 \\ 0 \\ 12 \end{bmatrix} \qquad \text{ and } \qquad T\lp \begin{bmatrix} 0 \\ 1 \end{bmatrix} \rp = \begin{bmatrix} 0 \\ 3 \\ 7 \end{bmatrix} \end{equation*}
    If this were a linear transformation, then property (ii) would mean that
    \begin{equation*} T\lp \begin{bmatrix} 1 \\ 1 \end{bmatrix} \rp = T\lp \begin{bmatrix} 1 \\ 0 \end{bmatrix} \rp + T\lp \begin{bmatrix} 0 \\ 1 \end{bmatrix} \rp = \begin{bmatrix} 2 \\ 0 \\ 12 \end{bmatrix} + \begin{bmatrix} 0 \\ 3 \\ 7 \end{bmatrix} = \begin{bmatrix} 2 \\ 3 \\ 19 \end{bmatrix} \end{equation*}
    But in fact,
    \begin{equation*} T\lp \begin{bmatrix} 1 \\ 1 \end{bmatrix} \rp = \begin{bmatrix} 2 \\ 3 \\ 12 \end{bmatrix} \end{equation*}
    Oopsie!
    🔗
    🔗

    Example 1.7.8. An example.

    The function \(T: \R^2 \to \R^2\) given by
    \begin{equation*} T\lp \begin{bmatrix} x \\ y \end{bmatrix} \rp = \begin{bmatrix} 2x + 3y \\ 5x + 7y \end{bmatrix} \end{equation*}
    IS indeed a linear transformation!
    🔗
    Proof: We will verify the two properties in the definition of linear transformations.
    1. Suppose \(c\) is a scalar and \(\begin{bmatrix} x \\ y \end{bmatrix}\) is a vector in \(\R^2\text{.}\) Then
      \begin{align*} T\lp c\begin{bmatrix} x \\ y \end{bmatrix} \rp = T\lp \begin{bmatrix} cx \\ cy \end{bmatrix} \rp \amp= \begin{bmatrix} 2(cx) + 3(cy) \\ 5(cx) + 7(cy) \end{bmatrix} \\ \amp= \begin{bmatrix} c(2x + 3y) \\ c(5x + 7y) \end{bmatrix} \\ \amp= c\begin{bmatrix} 2x + 3y \\ 5x + 7y \end{bmatrix} \\ \amp= cT\lp \begin{bmatrix} x \\ y \end{bmatrix} \rp \end{align*}
      🔗
      🔗
    2. Suppose \(\begin{bmatrix} x_1 \\ y_1 \end{bmatrix}\) and \(\begin{bmatrix} x_2 \\ y_2 \end{bmatrix}\) are two vectors in \(\R^2\text{.}\) Then
      \begin{align*} T\lp \begin{bmatrix} x_1 \\ y_1 \end{bmatrix} + \begin{bmatrix} x_2 \\ y_2 \end{bmatrix} \rp = T\lp \begin{bmatrix} x_1 + x_2 \\ y_1 + y_2 \end{bmatrix} \rp \amp= \begin{bmatrix} 2(x_1 + x_2) + 3(y_1 + y_2) \\ 5(x_1 + x_2) + 7(y_1 + y_2) \end{bmatrix} \\ \amp= \begin{bmatrix} (2x_1 + 3y_1) + (2x_2 + 3y_2) \\ (5x_1 + 7y_1) + (5x_2 + 7y_2) \end{bmatrix} \\ \amp= \begin{bmatrix} 2x_1 + 3y_1 \\ 5x_1 + 7y_1 \end{bmatrix} + \begin{bmatrix} 2x_2 + 3y_2 \\ 5x_2 + 7y_2 \end{bmatrix} \\ \amp= T\lp \begin{bmatrix} x_1 \\ y_1 \end{bmatrix} \rp + T\lp \begin{bmatrix} x_2 \\ y_2 \end{bmatrix} \rp \end{align*}
      🔗
      🔗
    Since both properties hold, we can conclude that \(T\) IS indeed a linear transformation!
    🔗
    Note: In this example, observe that the coefficients do not contribute to the proof at all (so the proof would work exactly the same even if we had different coefficients). Using the same proof, we can show that the function \(T: \R^2 \to \R^2\) given by
    \begin{equation*} T\lp \begin{bmatrix} x \\ y \end{bmatrix} \rp = \begin{bmatrix} a_1x + b_1y \\ a_2x + b_2y \end{bmatrix} \end{equation*}
    is a linear transformation for any choice of scalars \(a_1, b_1, a_2, b_2\text{.}\)
    🔗
    🔗
    Using linearity, we can quickly obtain the following theorem:
    🔗

    Theorem 1.7.9.

    If \(T\) is a linear transformation, then \(T\lp \v{0} \rp = \v{0}\)
    🔗
    🔗

    Very quick proof of this theorem.
    \(T\lp \v{0} \rp = T\lp 0 \v{u} \rp = 0 T\lp \v{u} \rp = \v{0}\)
    🔗
    🔗
    This is an important observation later when we investigate the idea of subspaces (spoiler: the kernel is a subspace and one of the reasons is because of this!)
    🔗
    The linearity property of linear transformations makes it possible to "plug in" a vector even if we don’t know the rule. That is, we can evaluate linear transformations using linear combinations.
    🔗

    Example 1.7.10.

    Suppose \(T: \R^3 \to \R^2\) is a linear transformation (but we don’t know the rule) and we do know that
    \begin{equation*} T \lp \begin{bmatrix} 1 \\ 2 \\ -1 \end{bmatrix} \rp = \begin{bmatrix} 2 \\ 0 \end{bmatrix} \qquad \text{ and } \qquad T \lp \begin{bmatrix} 1 \\ 2 \\ 3 \end{bmatrix} \rp = \begin{bmatrix} 1 \\ -4 \end{bmatrix} \end{equation*}
    Let’s say we want to evaluate \(T \lp \begin{bmatrix} 5 \\ 10 \\ 3 \end{bmatrix} \rp\text{.}\)
    🔗
    How do we do that?
    🔗
    Notice that
    \begin{equation*} \begin{bmatrix} 5 \\ 10 \\ 3 \end{bmatrix} = 3\begin{bmatrix} 1 \\ 2 \\ -1 \end{bmatrix} + 2\begin{bmatrix} 1 \\ 2 \\ 3 \end{bmatrix} \end{equation*}
    So property (i) and (ii) of linear transformations tell us that
    \begin{align*} T \lp \begin{bmatrix} 5 \\ 10 \\ 3 \end{bmatrix} \rp \amp= T \lp 3\begin{bmatrix} 1 \\ 2 \\ -1 \end{bmatrix} + 2\begin{bmatrix} 1 \\ 2 \\ 3 \end{bmatrix} \rp \\ \amp= 3T \lp \begin{bmatrix} 1 \\ 2 \\ -1 \end{bmatrix} \rp + 2T \lp \begin{bmatrix} 1 \\ 2 \\ 3 \end{bmatrix} \rp \\ \amp= 3\begin{bmatrix} 2 \\ 0 \end{bmatrix} + 2\begin{bmatrix} 1 \\ -4 \end{bmatrix} \\ \amp= \begin{bmatrix} 8 \\ -8 \end{bmatrix} \end{align*}
    🔗
    🔗
    This is a very important kind of problem! Not only does it show up on exams commonly, it will also be the key to finding the matrix for a linear transformation which is one of the most important things in linear algebra!
    🔗

    Checkpoint 1.7.11.

    Suppose \(T: \R^3 \to \R^2\) is a linear transformation and we know that
    \begin{equation*} T \lp \begin{bmatrix} 1 \\ 1 \\ 3 \end{bmatrix} \rp = \begin{bmatrix} 2 \\ 1 \end{bmatrix} \qquad \text{ and } \qquad T \lp \begin{bmatrix} 2 \\ 1 \\ 0 \end{bmatrix} \rp = \begin{bmatrix} -2 \\ 3 \end{bmatrix} \end{equation*}
    Find \(T \lp \begin{bmatrix} 1 \\ -1 \\ -9 \end{bmatrix} \rp\text{.}\)
    🔗
    Solution.
    First, we need to express \(\begin{bmatrix} 1 \\ -1 \\ -9 \end{bmatrix}\) as a linear combination of the two input vectors we know. We are looking for scalars \(c_1\) and \(c_2\) such that:
    \begin{equation*} c_1\begin{bmatrix} 1 \\ 1 \\ 3 \end{bmatrix} + c_2\begin{bmatrix} 2 \\ 1 \\ 0 \end{bmatrix} = \begin{bmatrix} 1 \\ -1 \\ -9 \end{bmatrix} \end{equation*}
    🔗
    Looking at the third row, we immediately see that \(3c_1 + 0 = -9\text{,}\) which means \(c_1 = -3\text{.}\)
    🔗
    Substituting \(c_1 = -3\) into the second row gives us \(-3(1) + c_2(1) = -1\text{,}\) which means \(c_2 = 2\text{.}\) Let’s quickly verify with the first row: \(-3(1) + 2(2) = -3 + 4 = 1\text{.}\) It works!
    🔗
    Now we can use the linearity properties to evaluate the transformation:
    \begin{align*} T\lp \begin{bmatrix} 1 \\ -1 \\ -9 \end{bmatrix} \rp \amp= T\lp -3\begin{bmatrix} 1 \\ 1 \\ 3 \end{bmatrix} + 2\begin{bmatrix} 2 \\ 1 \\ 0 \end{bmatrix} \rp \\ \amp= -3T\lp \begin{bmatrix} 1 \\ 1 \\ 3 \end{bmatrix} \rp + 2T\lp \begin{bmatrix} 2 \\ 1 \\ 0 \end{bmatrix} \rp \\ \amp= -3\begin{bmatrix} 2 \\ 1 \end{bmatrix} + 2\begin{bmatrix} -2 \\ 3 \end{bmatrix} \\ \amp= \begin{bmatrix} -6 \\ -3 \end{bmatrix} + \begin{bmatrix} -4 \\ 6 \end{bmatrix} = \begin{bmatrix} -10 \\ 3 \end{bmatrix} \end{align*}
    🔗
    🔗
    🔗
    What about the other way around? What if we are given the image of some linear transformation and we are trying to find the input?
    🔗
    Then instead of "evaluating", we will be "solving" the matrix equation. We are all experts in row reduction so this shouldn’t be a problem!
    🔗

    Example 1.7.12.

    Let \(A = \begin{bmatrix} 1 \amp 3 \\ 9 \amp -1 \\ -2 \amp 5 \end{bmatrix}\text{.}\) We define \(T: \R^2 \to \R^3\) by \(T(\v{x}) = A\v{x}\text{.}\) Now we want to find \(\v{x}\in \R^2\) such that \(T\lp \v{x} \rp = \begin{bmatrix} 3 \\ -29 \\ 16 \end{bmatrix}\text{.}\)
    🔗
    If we plug in everything we know into the transformation, then we will get the following matrix equation:
    \begin{equation*} \begin{bmatrix} 1 \amp 3 \\ 9 \amp -1 \\ -2 \amp 5 \end{bmatrix} \v{x} = \begin{bmatrix} 3 \\ -29 \\ 16 \end{bmatrix} \end{equation*}
    Converting this into an augmented matrix, we obtain
    \begin{equation*} \begin{bmatrix} 1 \amp 3 \amp \mid \amp 3 \\ 9 \amp -1 \amp \mid \amp -29 \\ -2 \amp 5 \amp \mid \amp 16 \end{bmatrix} \end{equation*}
    🔗
    Let’s apply our Gaussian Elimination skills to solve for \(\v{x}\text{:}\)
    \begin{align*} \begin{bmatrix} 1 \amp 3 \amp \mid \amp 3 \\ 9 \amp -1 \amp \mid \amp -29 \\ -2 \amp 5 \amp \mid \amp 16 \end{bmatrix} \amp\sim \begin{bmatrix} 1 \amp 3 \amp \mid \amp 3 \\ 0 \amp -28 \amp \mid \amp -56 \\ 0 \amp 11 \amp \mid \amp 22 \end{bmatrix} \amp\amp \begin{matrix} -9\text{Row 1} + \text{Row 2} \\ 2\text{Row 1} + \text{Row 3} \end{matrix} \\ \amp\sim \begin{bmatrix} 1 \amp 3 \amp \mid \amp 3 \\ 0 \amp 1 \amp \mid \amp 2 \\ 0 \amp 1 \amp \mid \amp 2 \end{bmatrix} \amp\amp \begin{matrix} -\frac{1}{28}\text{Row 2} \\ \frac{1}{11}\text{Row 3} \end{matrix} \\ \amp\sim \begin{bmatrix} 1 \amp 0 \amp \mid \amp -3 \\ 0 \amp 1 \amp \mid \amp 2 \\ 0 \amp 0 \amp \mid \amp 0 \end{bmatrix} \amp\amp \begin{matrix} -3\text{Row 2} + \text{Row 1} \\ -\text{Row 2} + \text{Row 3} \end{matrix} \end{align*}
    🔗
    Translating the reduced row echelon form back into an equation, we get \(x_1 = -3\) and \(x_2 = 2\text{.}\) Therefore, the input vector we were looking for is:
    \begin{equation*} \v{x} = \begin{bmatrix} -3 \\ 2 \end{bmatrix} \end{equation*}
